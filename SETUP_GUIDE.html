SyncDrop AI — Setup Guide

Everything required to configure Supabase, build the Android APK, sideload it, and sign in successfully. Follow the steps in order the first time; after that, only the build steps (5–6) are needed when code changes.

1 Create the Supabase project

Skip this if you already have a project — note its name for the next steps.

  1. Go to supabase.com and sign in.
  2. Click New project, choose an organization, name it (e.g. syncdrop-ai), set a database password, and pick a region close to you.
  3. Wait for provisioning to finish (a minute or two).

2 Get your API credentials

These go straight into .env in step 5.

  1. In the project dashboard, open Project Settings → API.
  2. Copy the Project URL (looks like https://<ref>.supabase.co) — this is VITE_SUPABASE_URL.
  3. Copy the anon / public key (NOT the service_role key) — this is VITE_SUPABASE_ANON_KEY.
Never use the service_role key in the app. It bypasses Row Level Security entirely. Only the anon/public key belongs in a client app.

3 Set up storage bucket & database schema

  1. Go to Storage in the dashboard and create a new private bucket named exactly files (matches VITE_SUPABASE_BUCKET).
  2. Go to SQL Editor, open the project's supabase/schema.sql file in your repo, and run its contents to create the public.files table and Row Level Security policies.
  3. Optional: deploy the supabase/functions/suggest-filename Edge Function if you want AI-generated filenames (requires an Anthropic API key set as a function secret).

4 Configure auth redirect URLs (required for mobile sign-in)

Without this step, tapping the magic-link in your email will fail on the phone — Supabase only redirects to URLs it has explicitly allow-listed.

  1. Go to Authentication → URL Configuration.
  2. Under Redirect URLs, add: 
    com.syncdrop.ai://login-callback
  3. Save.
Why this is needed The Android app registers a custom URL scheme (com.syncdrop.ai://login-callback) so that tapping the sign-in link in Gmail/your email app opens SyncDrop AI directly and completes sign-in — instead of trying (and failing) to open http://localhost:3000 in a browser.

5 Create your .env file

Create a file named .env in the project root (same folder as package.json). It is gitignored — never commit it.

VITE_SUPABASE_URL=https://<your-project-ref>.supabase.co
VITE_SUPABASE_ANON_KEY=<your-anon-public-key>
VITE_SUPABASE_BUCKET=files
SUPABASE_FUNCTIONS_URL=https://<your-project-ref>.supabase.co/functions/v1
ANTHROPIC_MODEL=claude-opus-4-8
VariableRequiredUsed for
VITE_SUPABASE_URLYesConnecting the app to your Supabase project
VITE_SUPABASE_ANON_KEYYesPublic client auth/storage access
VITE_SUPABASE_BUCKETNo (defaults to files)Storage bucket name
SUPABASE_FUNCTIONS_URLNoAI filename suggestion Edge Function
ANTHROPIC_MODELNoModel used by the Edge Function
Vite inlines these at build time, not runtime. Any time you change .env, you must rebuild (step 6) — restarting the app alone does nothing.

6 Build the web app and Android APK

Run from the project root. Requires Node.js, the Android SDK, and a Java 21 JDK (Capacitor's Android module needs it specifically — Java 11/17 will fail the Gradle build).

# 1. Install dependencies (first time only)
npm install

# 2. Build the web bundle — bakes in your .env values
npm run build

# 3. Copy the build into the native Android project
npx cap sync android

# 4. Build the debug APK
cd android
./gradlew.bat assembleDebug   # Windows
# or ./gradlew assembleDebug  # macOS/Linux
Java version If Gradle fails with invalid source release: 21 or requires Java 17, point JAVA_HOME at a JDK 21 install before running Gradle, e.g.: 
export JAVA_HOME="/c/Program Files/Microsoft/jdk-21.0.11.10-hotspot"
export PATH="$JAVA_HOME/bin:$PATH"

The finished APK is written to:

android/app/build/outputs/apk/debug/app-debug.apk

Copy it into builds/ for safekeeping if you want a stable path to sideload from.

7 Install the APK on your phone

  1. If SyncDrop AI is already installed from a previous build, uninstall it first — debug-signed APKs from different build runs can have signature mismatches that block updates.
  2. Transfer app-debug.apk to your phone via one of:
    • ADB (phone connected via USB with USB debugging enabled): adb install -r app-debug.apk
    • Cloud drive / email-to-self / USB drag-and-drop, then open it from a file manager app
  3. When prompted, allow "install unknown apps" for whichever app you used to open the file.

8 Signing in

  1. Open SyncDrop AI on your phone.
  2. Enter your email address and tap Send link.
  3. Open the email on the same phone and tap the sign-in link.
  4. Android should offer to open it with SyncDrop AI (or open it automatically) — this signs you in and returns you to the app.
Why you can't just click the link in a browser The link redirects to a Supabase auth callback URL. We've configured that callback to be a custom app scheme (com.syncdrop.ai://login-callback) rather than a website, so tapping it hands control straight to the installed app instead of a browser.

9 Optional: enable typed OTP codes

The app also has a fallback "enter the code from your email" field, for cases where tapping a link is inconvenient. This requires extra Supabase setup and is not required — the deep-link flow in step 8 works without it.

  1. Supabase only lets you edit email template content (to insert the raw {{ .Token }} code) once a custom SMTP provider is connected. Free options include Resend and Brevo.
  2. Set it up under Project Settings → Authentication → SMTP Settings.
  3. Then go to Authentication → Email Templates → Magic Link and add {{ .Token }} somewhere in the body, e.g.: 
    Or enter this code in the app: <strong>{{ .Token }}</strong>
  4. Save. Future sign-in emails will include a 6-digit code you can type directly into the app.

10 Troubleshooting

SymptomCauseFix
"Supabase not configured" banner .env missing/empty, or app built before .env was filled in Fill in .env (step 5), then rebuild (step 6) — Vite inlines values at build time
Link opens browser, says site unreachable / localhost error Redirect URL not allow-listed, or installed APK predates the deep-link fix Complete step 4, then rebuild and reinstall the APK
"email rate limit exceeded" Supabase's default email service caps sign-in emails per hour Wait ~1 hour, or set up custom SMTP for higher limits
APK won't install / update over old version Debug signing mismatch between build runs Uninstall the old app first, then install the new APK
Gradle build fails: "Android Gradle plugin requires Java 17" or "invalid source release: 21" Wrong JAVA_HOME Point JAVA_HOME at a JDK 21 install before running Gradle (step 6)
